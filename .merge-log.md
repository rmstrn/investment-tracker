## Admin bypass policy

Admin-merge via `gh pr merge --admin` is acceptable when ALL three conditions are met:
1. Local lint/typecheck/build/test are green
2. The only GitHub blocker is either (a) pre-existing unrelated failure on main, or (b) UNSTABLE state after force-push rebase
3. The admin-merge is logged in .merge-log.md with reason

The same principle applies to local `--no-verify` on `git push` when a
pre-push lefthook fails on a pre-existing unrelated issue (e.g. the
scaffold `py-mypy` failure in `apps/ai`) — log the bypass in the PR
description or merge-log entry.
