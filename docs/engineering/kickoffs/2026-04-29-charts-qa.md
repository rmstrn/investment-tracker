## 1. Why this slice

A chart subsystem fails silently in three ways: (a) the parser accepts malformed payloads and the renderer crashes downstream, (b) a theme regression turns a forest-jade gain into a bronze loss without anyone noticing, (c) a Lane-A guardrail (info-only chip, projection caption, staleness banner) gets removed during refactor and the product silently starts looking like advice. Unit tests catch (a). Visual regression catches (b). Runtime guardrail tests catch (c).

This slice ships all three layers in one coordinated test suite that pins chart correctness for the indefinite future. It is the safety net that lets us merge subsequent chart changes (new variant additions, theme tweaks, Recharts upgrades) without burning a multi-agent review every time.

## 5. Scope

### IN scope — three test layers

**Layer A — Parser tier (Vitest, fast, no DOM):**

- For each of 10 MVP chart kinds: a positive test (valid payload parses successfully and produces the expected discriminated-union variant). Scatter MVP-union test is negative-only (see below).
- For each of 3 Risk Flags from CHARTS_SPEC §5.2: a negative test (payload violating that flag is rejected by the parser; assert on the Zod issue path so future schema refactors don't accidentally weaken the check).
- **Δ1 — `MetaFinancialAggregate` sum-to-total tests:** for Donut / Treemap / StackedBar, one positive (sum within tolerance), one boundary (sum at exact tolerance edge), one negative (sum outside tolerance) per chart. Validates the shared mixin once and asserts the same invariant fires on all three consumers.
- **Δ2 — Waterfall conservation block-merge fixtures:** canonical `WaterfallPayload` parses; intentionally-broken variants reject with `params.code === 'WATERFALL_CONSERVATION_VIOLATION'` (distinct from generic ZodError). Required broken variants: (a) start+Σ off by $5, (b) off by $5000, (c) signs reversed on fees, (d) missing start anchor, (e) missing end anchor. Tolerance boundary: $1.00 absolute passes, $1.01 fails. **This invariant is block-merge severity** — failing tests in this group must block PR merge identically to a `targetWeight` rejection regression.
- **Δ3 — Scatter MVP-union exclusion:** `parseChartEnvelope({ kind: 'scatter', ... })` returns `{ ok: false, ... }` because Scatter is not a member of the MVP `ChartPayload` discriminated union. Test asserts on the discriminator-mismatch issue path, NOT on a feature-flag — there is no MVP runtime flag for scatter.
- **T-8 — Treemap basis discriminator (3 tests):** (a) treemap with `dailyChangeBasis: 'local'` parses successfully; (b) treemap with `dailyChangeBasis: 'base'` parses successfully; (c) treemap omitting the field fails parse with Zod issue path `['dailyChangeBasis']`. See backend kickoff §5 + finance audit §9 Δc.
- For each CRITICAL or HIGH finding from finance audit §3 not already covered by Δ1/Δ2/Δ3/T-8: a negative test that the parser rejects the violating payload (e.g., OHLC ordering, calendar `Currency` required-not-defaulted). **Explicit exclusion: S-2** (scatter prescriptive-vocab regex) is NOT in this set — Δ3 makes it unreachable for MVP; tracked as TD for V2 re-activation per finance audit §9 Δa.
- Snapshot test of the discriminated-union TypeScript output for documentation stability.

**Layer B — Visual regression (Playwright, screenshot diff):**

- **10 MVP charts × 2 themes = 20 baseline screenshots** (Scatter excluded per Δ3), captured at a single fixed viewport (1280×800).
- Each baseline uses a realistic fixture from CHARTS_SPEC §5.3 example payloads (no synthetic «hello world» fixtures — the realism reduces false-positive surprise on real-data renders).
- Diff threshold: 0.1% pixel difference (anti-aliasing tolerance). Any regression above threshold is a block-CI failure.
- Baselines stored in `tests/visual/baselines/<chart-kind>__<theme>.png`. Diff artefacts (when failures occur) saved to `tests/visual/__diffs__/` and uploaded as CI artefacts for review.
- A CLI `pnpm test:visual:update` regenerates all baselines (used after intentional visual changes). The regeneration step requires explicit local opt-in — never run automatically in CI.

**Layer C — Runtime guardrail tests (Vitest + Testing Library, DOM-aware):**

For each cross-cutting guardrail in finance audit §4, encode a runtime test:

1. **Info-only chip** — render a chart with `chip: 'info-only'` (or whatever the guardrail field is named per architect ADR); assert the chip is rendered with the «information only» label visible to assistive tech (`getByRole('status')` or similar). Render the same chart WITHOUT the chip flag; assert no chip is rendered.
2. **Mandatory caption** — render any chart that has a finance-mandated caption (e.g., treemap intra-day with staleness banner, drift-bar with rebalancing-context caption); assert the caption is rendered above the chart, not below.
3. **Projection rendering** — render a chart with a `projection` field set; assert the projection renders dashed (not solid) and a disclaimer caption is present.
4. **Currency formatting** — render a chart with `currency: 'EUR'`; assert tooltip values use locale-aware EUR formatting; render with `currency: 'USD'`; assert USD. (Architecture per architect ADR.)
5. **Lane-A vocabulary** — render a chart whose payload accidentally contains prescriptive vocabulary in `meta.factualCaption` (e.g., «you should rebalance»); assert the regex layer (per finance audit cross-cutting recommendation) catches it OR the parser rejected it earlier (preferred). The test verifies the defence, regardless of which layer wins.

For per-chart guardrails (e.g., candlestick T3 gate, calendar V2 paid-data gate per finance audit findings):

6. **Scatter MVP exclusion** — covered in Layer A (parser tier) per Δ3, NOT a runtime gate. AI-emitted `kind: 'scatter'` fails at `parseChartEnvelope` boundary; renderer never receives it. Layer C does not need a redundant runtime test for scatter; cite Layer A coverage in PR description.
7. **Candlestick T3 gate** — attempt to render a `kind: 'candlestick'` payload; assert the demo-block path is absent in the showcase route (Candlestick has no demo per FE kickoff scope) and that direct render via `LazyCandlestick` succeeds when given a valid payload (the gate is showcase-presence, not parse-time).
8. **Calendar V2 gate** — attempt to render a `kind: 'calendar'` configured with paid-data `eventType: 'earnings'` or `'news'`; assert parser rejection at Layer A; if a malformed payload reaches the renderer, `ChartError` state is shown (no silent skeleton fallback).

### OUT of scope (deferred or owned elsewhere)

- E2E test of the AI agent → chat → chart-render flow. Owned by chat-flow QA, not this slice.
- Performance benchmarks (render time, FPS during animation). Tracked as a follow-on TD; not a block for chart MVP.
- Cross-browser visual regression (Firefox / Safari). MVP locks Chromium baselines only. Cross-browser is a follow-on slice.
- Accessibility audit (axe-core). Tracked separately under the accessibility kickoff path; do NOT bundle here.
- Mobile-viewport visual baselines. MVP locks 1280×800 only; mobile baselines deferred.

## 6. Implementation plan

1. **Setup** — verify Playwright + Vitest are configured for the new test paths. Add `tests/visual/` directory; add `pnpm test:visual` and `pnpm test:visual:update` scripts.
2. **Layer A — Parser tests** — write before FE merges. They test backend's schema directly; do not need FE rendering. Land first; provides confidence the contract is locked.
3. **Layer C — Runtime guardrails** — write second; can run against FE's components in jsdom. Land before visual regression so any guardrail bug is caught before screenshot baselines lock the wrong UI.
4. **Layer B — Visual baselines** — write last; capture only after Layer C confirms guardrails are correct. Otherwise visual baselines lock incorrect UI and we pay for the mistake on every future chart change.
5. **CI wiring** — three jobs: `test:parser`, `test:guardrails`, `test:visual`. Parser job runs on every PR; visual job runs only on PRs that touch `packages/ui/src/charts/**` or `packages/design-tokens/**` (via path filter to avoid pointless reruns).
