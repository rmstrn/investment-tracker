**Rule:** target 1–2 syllables. 3-syllable only if phonetic flow is obviously smooth (Linear is edge case because «lin-yer» often collapses to 2).

### Letter count

- Range: 3–7 letters
- Median: 6
- Mean: 5.5
- Tight cluster at 5–6 letters

**Rule:** target 3–7 letters. 6 is the sweet spot. 8+ is a drift warning.

### Onset class

| Class | % | References |
|---|---|---|
| Plosive (/k/ /b/ /p/ /t/ /d/) | 33% | Canva, Claude /kl/, Cursor, Stripe /str/ |
| Fricative (/f/ /v/ /s/ /ʃ/) | 25% | Figma, Framer /fr/, Vercel |
| Liquid (/l/ /r/) | 25% | Linear, Loom, Raycast |
| Nasal (/n/ /m/) | 8% | Notion |
| Vowel | 8% | Arc |

**Dominant:** hard-edged onsets (plosive + fricative + clusters involving them) = **58%**. Liquid secondary at 25%. Nasal and vowel minority.

**Rule:** strongly prefer plosive or fricative onsets (incl. plosive-liquid and fricative-liquid clusters — /kl/, /fr/, /str/). Liquid onsets acceptable but not the center of gravity. Nasal and vowel onsets are minority and should not dominate a generation batch.

### Close class (final phoneme)

| Class | % | References |
|---|---|---|
| Plosive (/k/ /d/ /p/ /t/) | 33% | Claude, Arc, Raycast, Stripe |
| Liquid tail (/r/ /l/) | 33% | Linear, Cursor, Framer, Vercel |
| Nasal (/n/ /m/) | 17% | Notion, Loom |
| Open vowel (/a/) | 17% | Canva, Figma |

**Dominant:** bimodal. Two equally-weighted signatures — crisp plosive-close (Arc, Stripe) OR liquid-tail close (Framer, Cursor).

**Rule:** target plosive-close or liquid-tail close. Nasal-close is acceptable minority. Pure open-vowel close (Canva/Figma shape) works in 2-syl real-word-shadow cases but is a minority pattern — don't overdose.

### Body vowel tendency

- Closed/short vowels dominant: short-i (Figma, Linear), short-u (Cursor), short-e (Vercel), short-a (Canva close, Raycast second syl)
- Long-o common (Claude, Notion, Loom)
- Long-a sparse but present (Framer, Raycast first syl)
- Long-i/diphthong rare (Stripe only — the outlier)

**Rule:** short/closed vowels are the natural center. Long-o is acceptable. Long-i/diphthong shapes read as fintech-aggressive (Stripe is the tolerance edge) — use sparingly.

## Tone anchor (operational 5-criterion rubric 0–10)

Re-anchored against the 12 positives (replacing R16's borrowed Stripe/Brex/Mercury rubric). Each criterion has operational definition + 0/5/10 anchors drawn from the reference set.

### C1 — Consonant-Edge

**Definition:** Presence of crisp, compositor-hard onset and/or close phoneme. Measures phonetic decisiveness vs softness.

- **10 (max edge):** Stripe (/str/ + /p/), Arc (vowel + /k/), Claude (/kl/ + /d/), Raycast (/r/ + /t/)
- **5 (mid):** Figma (/f/ + open /a/), Canva (/k/ + open /a/), Cursor (/k/ + /r/)
- **0 (min edge):** Teven (soft /t/ + nasal close — the negative anchor), Shunkan (soft fricative + nasal)

**Target score for Provedo candidates:** **6–9.** Edge is load-bearing; don't go below 5.

### C2 — Decisive-Compression

**Definition:** Name lands in ≤2 syllables and ≤7 letters without feeling truncated. Measures compactness discipline.

- **10 (max compression):** Arc (1 syl / 3 letters), Loom (1 syl / 4 letters), Claude (1 syl / 6 letters), Stripe (1 syl / 6 letters)
- **5 (mid):** Figma (2 syl / 5 letters), Canva (2 syl / 5 letters), Notion (2 syl / 6 letters), Cursor (2 syl / 6 letters)
- **0 (min compression):** Noesis (3 syl / 6 letters — the negative anchor, drifts academic)

**Target score:** **7–10.** Round 17's compound names (Keelmark, Holdform) score ~5 here — that's the tolerance floor. Don't go below.

### C3 — Register-Fit (re-anchored: «modern AI/tech-tool 2020s»)

**Definition:** Does the name read as if it could launch tomorrow as a new modern software tool? Replaces R16's «fintech-infrastructure-schema» anchor (which was borrowed, not PO's taste).

- **10 (perfect fit):** Figma, Notion, Linear, Arc, Raycast, Cursor, Framer, Vercel — the native cluster
- **5 (adjacent fit):** Stripe (fintech crossover, but same tech-tool design register), Claude (AI-assistant crossover), Canva (consumer-design crossover)
- **0 (wrong register):** Noesis (academic-philosophy), Bodh (mystical), Shunkan (exotic-warm), Teven (companion-warm)

**Target score:** **8–10.** This is the single most load-bearing criterion for Round 18 — it's the criterion that failed in 17 previous rounds when anchor was borrowed.

### C4 — Invented-But-Rooted

**Definition:** Even when the name is coined or re-purposed, the phonotactics and shape feel like they «could be» a real English word. Prevents slug-territory.

- **10 (max rooted):** Canva (from canvas), Framer (from frame+er), Figma (from figura — coined but rooted), Stripe (real word, shadow repurposed)
- **5 (mid):** Vercel (coined, but phonotactically English-feeling), Raycast (compound from real English roots)
- **0 (min rooted — slug territory):** Lerx (the negative anchor — pure consonant-cluster slug, zero root)

**Target score:** **7–10.** R17 «Nevor» scores ~6–7 here (Latin root present, but shadow is invisible to non-Latinist English speaker). Round 18 should aim higher — root should be readable to native English speaker on first parse.

### C5 — Verb-Potential

**Definition:** Does the name support verb / action usage in user-facing copy? «I Slacked him», «I Ubered it», «Let me Notion that». Measures functional-grammar adoption runway.

- **10 (max verb-potential):** Loom («Loom it»), Arc (native English verb root), Cursor (native English noun that verbs cleanly), Framer (maker suffix is inherently verb-adjacent)
- **5 (mid):** Canva, Figma, Notion, Linear, Vercel — all usable as products-of-action but don't verb naturally in speech
- **0 (min verb-potential):** Noesis (academic abstract, doesn't verb), Bodh (mystical noun, doesn't verb)

**Target score:** **5–9.** Not every winner verbs (Figma doesn't, Notion barely does, and they're both successful), but this is the criterion that captures Magician-archetype «tool-in-hand» feel. Below 5 is a warning.

### C6 — Persona-fit (added 2026-04-25; emergent R26)

**Target score for Provedo candidates:** **6–10.** PO's R26 agent-persona pivot put weight on this. Below 5 = product is wearing a name, not embodying it.

## Provedo brand-name usage rules (mandatory for all downstream agents)

These rules are non-negotiable and apply to: content-lead, product-designer, frontend-engineer (UI strings), AI-persona prompt authors, and any external/contractor copy writer touching the brand.

### Capitalization

- **English / general:** «**Provedo**» — capital P, lowercase remainder. Never «PROVEDO», never «provedo», never «ProVedo».
- **Russian (Latin-script primary):** «**Provedo**» — same Latin-script capitalization. Default RU copy uses Latin script.
- **Russian (Cyrillic with stress mark, phonetic-guidance contexts):** «**Прове́до**» — capital П, acute stress mark on «е» to lock pronunciation. Use ONLY in first-touch contexts (about-page, pronunciation guide, brand FAQ). Do NOT use stress mark in body copy.
- **In headlines / hero / display type:** capitalization stays «Provedo» (do NOT all-caps-style for emphasis; the Sage/Magician archetype rejects shouting).

### Pronunciation

- **English:** /proh-VEH-doh/ (stress on second syllable, three open vowels)
- **Russian:** /прове́до/ (stress on «е», phonetically nearly identical to «проведу»)
- **First-touch pronunciation copy block (recommended):** «Provedo /proh-VEH-doh/ — from Italian *provedere*, "to provide for, to foresee". In Russian: прове́до — phonetically «проведу», "I will lead through".»

### Spelling variants

- Allowed: Provedo (default), Прове́до (RU phonetic-guidance only)
- Banned: Provido, Providere, Provedere (these are Italian/Latin source-roots, not the brand), Pro Vedo, Pro-vedo, ProVedo, PROVEDO

### Bilingual wordplay (RU)

RU copy can leverage the native verb-form decode naturally. The following collocations are **encouraged** because they reinforce the JTBD claim without forcing wordplay:

- «Provedo проведёт через твой портфель»
- «Provedo проведёт через дивиденды / просадку / волатильность»
- «Provedo, проведи меня через…»
- «Доверь Provedo провести тебя через…»
- «Меня сейчас Provedo проведёт»

These read as natural Russian, not as forced naming-puns. They are unique-to-Provedo copy assets (no other 32-round candidate could conjugate this way) and should be treated as primary RU brand-narrative slots.

### Bilingual wordplay (EN) — restricted

EN copy MUST NOT force verb-form usage of «Provedo». The following are **banned**:

- «Let me Provedo that» (forced — `provide` is already an EN verb)
- «I Provedo'd it» (does not English-conjugate)
- «Provedo it!» as call-to-action

Acceptable EN agent self-reference (where forced is unavoidable):
- «Ask Provedo» (chat-input button — at upper-edge of comfortable button-shape repetition; monitor for length friction)
- «Provedo noticed…» / «Provedo surfaces…» / «Provedo shows…» (observation-coded, Lane A clean)

### Verb-form summary

- RU: «провести / проводить» works natively as the brand's grammatical verb. Use freely.
- EN: brand stays a noun-as-named-agent. Do not English-verb the name. Avoid «to Provedo» as call-to-action pattern.

### First-touch etymology block (for landing / about / FAQ)

When the user encounters Provedo for the first time and an etymology touch is appropriate, use this canonical block (do NOT paraphrase):

> «Provedo — from Italian *provedere*, «to provide for, to foresee». For Russian-speakers, the name natively reads as «прове́до» — phonetically «проведу», "I will lead through". The product is the same in either reading: a second brain that leads you through your portfolio.»

This block is the load-bearing first-touch artifact. It pre-empts the «как это относится к нашему проекту?» first-read question (the question that killed Lucri / Vala / Bogle in calibration) and resolves it in 3 sentences.

## Brand voice fingerprint — Provedo refined (post-lock)

Refines the archetype + register description with Provedo as the operational anchor.

### Sage-archetype tone (primary)

- **Foresight:** anticipating the user's portfolio question before they ask it. Not predictive (Lane A: no predictions of returns), but observational-pattern-anticipatory («Provedo noticed you tend to look at dividends on payment day — here's the dashboard ready for you»).
- **Observation:** describing what is, not prescribing what should be. «Provedo sees / surfaces / shows / cites» — never «Provedo recommends / advises / strategizes».
- **Calm clarity:** restrained tone (Notion-restrained lock 2026-04-24, H-096). No exclamation marks in agent output. No emoji in agent voice. No urgency-marketing register.

### Magician-archetype tone (primary modifier)

- **Pattern recognition:** «Provedo connects three trades you made across last six months and shows the underlying pattern.» — Magician-archetype tool-in-hand capability.
- **Hidden value surfacing:** «Provedo found a corporate action you missed in November.» — surfacing what was always there but invisible to user.
- **Foresight as Magician (NOT predictive):** «Provedo sees the rebalance window approaching for your retirement allocation.» — foresight = visibility-of-pattern, not market-prediction.

### Everyman-archetype tone (modifier — accessibility layer)

- **Friendly approachable:** plain language, no finance-jargon when a normal word will do. «Your dividends» not «your investment-income-stream-flows».
- **Caring helper:** «Provedo is here to help you see what you have» — not «Provedo is your fiduciary guardian» (paternalistic, Lane A risk).
- **NOT corporate-cold:** no «Dear Customer», no «Our Premium Subscriber», no third-person-distance.

### Anti-tone registers (explicitly NOT Provedo)

- **NOT advisor-prescriptive:** «Buy this / sell that / rebalance into X» — directly violates Lane A. Provedo never voices these.
- **NOT profit-aggressive:** «Maximize your gains / beat the market / outperform» — wrong archetype (Outlaw / Hero), wrong audience pull (anti-ICP day-trader).
- **NOT surveillance-creepy:** «Provedo watches you / Provedo tracks your every move / Provedo analyzes your behavior» — anti-pattern surfaced in PO calibration trail; surveillance register destroys the Sage/Everyman caring-helper warmth. Use «sees / notices / surfaces» (active-by-the-tool) not «watches / tracks / monitors» (passive-on-the-user).
- **NOT insurance-vendor:** «Your Provedo provider says…» — US insurance-coded register from «provide» drift. Avoid.
- **NOT companion-warm-Sidekick:** Provedo is NOT a friend / pal / buddy / cute-mascot. Sage gravitas + Magician craft + Everyman accessibility, not pet-companion warmth.

### Persona introduction templates (canonical)

- **EN:** «Hi, I'm Provedo. I provide clarity on your portfolio. I show you what's there, surface what you've missed, and help you see ahead.»
- **RU:** «Привет, я Provedo. Проведу тебя через твой портфель — покажу что есть, замечу что упустил, помогу увидеть дальше.»

These are canonical anchor templates. Variations are fine; the verb-set («provide clarity / show / surface / see ahead» EN; «провести / показать / заметить / увидеть» RU) is the in-tone register baseline.

## Provedo EN copy guardrails (mandatory)

**Why these guardrails exist:** Italian *provedere* etymology in English reads as «provide», which has stable advisor-adjacent collocations («provide advice / recommendations / strategy / guidance»). These collocations would emerge naturally from PR / growth / contractor copy writers because the verb-stem pulls there. The pull crosses Lane A regulatory boundary (US SEC / EU MiFID II / UK FCA / Russia 39-ФЗ — investment-advice activity is regulated). Hard rules below prevent this drift mechanically without requiring case-by-case editorial judgment.

**Asymmetry:** RU side has NO equivalent gradient. «Провести / проводить» = «lead through / guide» = Sage-archetype clean in any RU collocation. RU copy needs no special guardrails. The 5-item rules apply EN-side only.

**Owners:** content-lead enforces these on every EN copy review; brand-voice-curator audits for emergent drift quarterly; AI-persona system-prompt (Item 2) is mandatory baseline for every Claude / GPT / future-AI persona prompt that speaks as Provedo.

### 1. Banned EN co-occurrence rule

«Provedo» NEVER appears in the same sentence as any of:
- advice / advise / advisory
- recommendation / recommend / recommendations
- strategy / strategic / strategist
- suggestion / suggest

Hard rule. No exceptions. Applies to: landing copy, hero, taglines, microcopy, email subjects + body, ads, social copy, blog, partnership content, press releases, AI agent output, button labels, error messages, onboarding flows, push notifications, system-prompts.

If a copy line wants to say one of these things, restructure the sentence so «Provedo» is not the subject of the verb. Example:
- BANNED: «Provedo recommends rebalancing your tech allocation.»
- ALLOWED: «Your tech allocation is up 47% YTD — here's the rebalance window for the quarterly review.» (no agent-voice attribution to recommendation; user receives observation, decides themselves)

### 3. EN verb-allowlist for «Provedo» as agent-subject

Allowed agent-subject collocations (Sage-clean):

- Provedo provides context
- Provedo provides observation
- Provedo provides foresight

Banned agent-subject collocations (advisor-drift):

- ~~Provedo provides recommendations~~

### 4. «Guidance» splitter rule

The word «guidance» is borderline in financial regulatory context (FCA «guidance» specifically, FINRA Rule 2210 «recommendation» framework). It sits one step away from advice on the gradient.

- **Avoid:** «Provedo provides guidance on rebalancing» / «Provedo provides guidance on tax decisions»
- **Prefer:** «Provedo provides clarity on your rebalance windows» / «Provedo brings clarity to tax-related events you've made this year»

Rule of thumb: if the copy line wants to say «Provedo provides guidance», substitute «clarity» or «observation» first; if those don't fit, restructure the sentence so Provedo is not the subject of «provide guidance» at all.
