# TD-099 Forbidden-Vocabulary Workshop — Legal Angle

**Author:** legal-advisor (internal SME, NOT licensed counsel)
**Date:** 2026-04-29
**Scope:** AI-emitted chart-string fields (`meta.title`, `meta.subtitle`, `meta.alt`, `CalendarEvent.description`)
**Boundary enforced:** Lane A (information / education only) — anything that crosses into Lane B (personalized investment advice) triggers SEC Advisers Act 1940, MiFID II Article 9, FCA COBS 9, and Russian 39-ФЗ regulatory exposure.

---

## CAVEAT (read first)

This list is **internal product-validation tooling**, not a substitute for jurisdiction-specific licensed counsel review. The blocklist reduces obvious leak-surface but does not create regulatory immunity. Before any production market launch (US / EU / UK), a licensed compliance attorney must verify (a) the list is sufficient and (b) downstream UI copy generated by these strings stays Lane-A compliant. Russia is currently out-of-scope per the 2026-04-23 lock; the RU column exists to protect the bilingual `*-ru.md` output paths and CIS-language users, not to authorize 39-ФЗ-regulated activity.

## Regulatory anchors (why each tier exists)

| Anchor | Trigger language pattern |
|---|---|
| SEC Advisers Act §202(a)(11) — publisher-exclusion lost when advice becomes «specific to a particular client» | Imperative second-person verbs + holding references |
| Lowe v. SEC (1985) | «Particularized» vs «impersonal» — pronouns of address + recommendation verbs |
| MiFID II Art. 4(1)(4) + ESMA Q&A on investment advice | «Personal recommendation» = advice presented as suitable for the recipient |
| FCA COBS 9 / Perimeter Guidance PERG 8.28 | «Advice» = opinion on the merits of buying/selling, especially when paired with suitability framing |
| SEC anti-touting (§17(b) Securities Act) | Promotional verbs implying inflated value without disclosure |
| 39-ФЗ ст. 6.1 (инвестиционное консультирование) | «Индивидуальная рекомендация» — adressed advice + securities reference |

- `sell` (imperative) — action directive
- `hold` (imperative) — action directive

## Tier 2 — permissible only with strict factual context

These tokens are common in legitimate financial reporting and **must NOT be blanket-blocked**. Block only when paired with first/second-person framing or future-tense subjective claims. Implementation suggestion: allow at lemma level, but flag when co-occurring with personal pronouns (`you`, `your`, `вы`, `ваш`) or future-without-source.

### English (Tier 2)

- `expect` / `expected` — OK: «dividend expected at $0.42 per share» (factual, sourced). NOT OK: «expected to outperform».
- `forecast` / `forecasted` — OK with attributed source («consensus forecast: $1.20 EPS»). NOT OK as bare claim.
- `project` / `projected` — same rule as forecast.
- `outperform` / `underperform` — OK as historical fact («outperformed S&P by 4% in Q3»). NOT OK as forward claim.
- `beat` / `miss` — OK as past earnings fact. NOT OK forward.
- `target` (price target) — OK only as third-party-attributed («analyst target: $X»). NOT OK as house view.
- `likely` / `probable` — generally restricted; require sourced statistical context.
- `trend` / `trending` — OK descriptive; NOT OK as predictive imperative.
- `signal` — OK technical-indicator factual («50-day MA crossed 200-day»). NOT OK as actionable.
- `opportunity` — borderline; restrict in `meta.subtitle` outright.

### Russian (Tier 2)

- `ожидается` / `ожидаемый` — OK факт («дивиденд ожидается $0.42»). NOT OK: «ожидается рост».
- `прогноз` / `прогнозируется` — OK с атрибуцией источника.
- `превзошёл` / `опередил` / `отстал` — OK исторически. NOT OK прогнозно.
- `превысил` / `недотянул` (по EPS) — OK факт.
- `цель` / `целевая цена` — OK только с атрибуцией третьей стороны.
- `вероятно` / `скорее всего` — ограничить; требует статистики со ссылкой.
- `тренд` / `тенденция` — OK описательно.
- `сигнал` — OK технический индикатор как факт.
- `возможность` — borderline; в `meta.subtitle` блокировать.

## Cross-cutting form rules

These are pattern-level, not token-level. Implementation suggestion: regex-based, applied AFTER Tier-1 token check.

1. **Imperative mood + financial action verb** — block all imperatives addressed to the user that reference buy/sell/hold/exit/enter/own. Cover both moods: English bare-imperative («buy NVDA») and Russian императив 2nd person.
2. **«You» / «your» / «вы» / «ваш» + position/holding/portfolio** — any second-person possessive linked to a security position is per se outside Lane A («your NVDA position should…»).
3. **Adverbs of future certainty** — block: `certainly`, `surely`, `definitely`, `undoubtedly`, `guaranteed`, `наверняка`, `точно`, `гарантированно`, `обязательно`, `безусловно` when adjacent to a forward-looking claim.
4. **Comparative-superlative + action** — block «best move», «top pick», «лучше всего», «оптимально», «наилучший выбор» when paired with a security or action verb.
5. **«We» / «our» + recommendation-verb** — first-person-plural advisor framing («we recommend», «мы советуем») — even if `recommend` is already Tier-1, block the bare «we / мы» + action-verb collocation as belt-and-suspenders.
6. **Question-form veiled advice** — «Should you buy NVDA?» / «Стоит ли продавать?» — block interrogative form of any Tier-1 verb stem.
7. **Time-pressure framing** — `now is the time`, `сейчас самое время`, `пока не поздно` — touting/urgency language; SEC anti-touting concern.

## Specific regulatory triggers — token mapping

### 1. SEC publisher-exclusion («general» prong)
Trigger verbs: any second-person imperative + any first-person-plural recommendation verb. The Lowe v. SEC «impersonal» test fails the moment copy addresses the user about their holdings. **Highest-priority blocks:** `recommend`, `advise`, `should`, `you should`, second-person imperatives.

### 2. MiFID II Article 9 / ESMA «personal recommendation»
ESMA's test: (a) presented as suitable, (b) based on personal circumstances, (c) concerns specific instrument. Tokens that flip (a): `recommend`, `suggest`, `prefer`, `favor`, `suitable`, `appropriate for you`. Block all suitability-coded English adjectives in chart-string fields too: `suitable`, `appropriate`, `right for you`.

### 3. FCA COBS 9.5 / PERG 8.28
«Advice on the merits» trigger verbs: `advise`, `recommend`, opinion-verbs paired with specific instruments. Same Tier-1 set covers.

### 4. 39-ФЗ / Russian — special tokens
Russian regulatory text uses «индивидуальная инвестиционная рекомендация» (ИИР). Beyond Tier-1 list:
- `ИИР` / `индивидуальная рекомендация` — block the term itself appearing in output
- `персональная рекомендация` — block
- `подходит вам` / `подходящий для вас` — suitability per ст. 6.1
- `с учётом ваших целей` — suitability framing

### 5. SEC anti-touting (§17(b))
Promotional / pump verbs:
- `skyrocket`, `moon`, `explode`, `surge` (forward) — block
- `взлетит`, `выстрелит`, `ракета`, `улетит` — block
- `must-own`, `must-have`, `обязательно держать` — block
- `hidden gem`, `скрытая жемчужина` — block

## Top-3 false-positive concerns (do NOT block)

1. **`provide` / `provedo`** — brand name collision. The verb «provide» is a common, factually-neutral English verb (a fund «provides exposure to», a dividend «provides $X»). Confirm regex word-boundaries handle the collision; the brand name `Provedo` in Russian is «проведу» (I will guide/lead) — must NOT trigger, even though `провед-` stem could naively look like a verb. **Implementation:** brand whitelist BEFORE blocklist scan.
2. **`accrue` / `settle` / `mature` / `vest`** — technical financial verbs. «Bond matures in 2027», «options vest over 4 years», «dividend accrues daily», «trade settles T+2» — all permissible factual statements. None are on Tier-1, but reviewers may instinctively want to block; confirm they remain free.
3. **`expect` / `forecast` / `target` / `ожидается` / `прогноз`** (already Tier-2) — these are essential to legitimate factual chart subtitles. The risk is over-blocking and producing sterile, useless chart copy that tanks UX. The Tier-2 contextual rules (require attribution / forbid forward-house-view) preserve utility.

Additional borderline-but-fine: `rise`, `fall`, `gain`, `lose`, `return`, `yield`, `compound`, `diversify` (descriptive), `hedge` (descriptive), `correlate`, `track`, `lag`, `lead`, `recover`. All factual market-mechanics vocabulary; do not list.

## Recommendation: review cadence

**Yes — quarterly cadence required**, with these triggers for ad-hoc earlier review:

- Any new market jurisdiction added to launch plan (e.g., adding APAC adds JFSA / MAS / SFC vocabulary)
- Any AI-model upgrade on the chart-string-emitting agent (new model = new generation patterns = new leak surfaces)
- Any regulator enforcement action published touching robo-advice / AI-generated investment content (ESMA, SEC, FCA periodically issue these — exa-search «robo-advice enforcement» quarterly)
- Any user-reported leak (a content-lead + legal triage queue should exist)

Quarterly review owners: legal-advisor (regulatory delta scan) + content-lead (false-positive log review) + finance-advisor (financial-vocabulary drift). 90-day cadence aligned with end-of-quarter regulator publication cycles.

- Advertising / endorsement disclosures — handled at marketing-copy layer, not chart-string layer
- Market-manipulation concerns — handled at data-source layer (broker-aggregator vetting)
